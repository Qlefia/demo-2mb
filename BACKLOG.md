# Technical Backlog (2mb CRM)

Items discovered during development that are **not blockers for the current task** but should be addressed later. This file is the single source of truth for tech debt, deferred refactoring, optimization opportunities, and pre-existing issues.

## How to use

- Each item has a **category**, a **short title**, and a **description**.
- Items are grouped by priority: **High** (before production), **Medium** (refactor / resilience / perf / security follow-ups), **Low** (cleanup, polish).
- **In progress** lists work that is only **partially** done (verified in code).
- When an item is **fully** resolved, move it to the **Done** section at the bottom with a date and a factual note.
- See [`.cursor/rules/backlog.mdc`](.cursor/rules/backlog.mdc) for the full discipline.

| 60  | Bug        | Studio settings autosave 409 + silent data loss on fast typing | 2026-05-21: Reproduced live with Playwright on the new bank-account detail page. Typing label + IBAN + bankName in quick succession produced 2× 409 in the console; after a reload, IBAN + bankName were **gone** even though label survived. Root cause: `useStudioProfileSync` fired a new debounced `mutation.mutate(...)` while the previous PUT was still in flight; the second PUT shipped a stale `expectedRevision` (the first push had not returned yet to advance `lastPushedRevision`) → server replied 409 → query was invalidated → `applyRemoteRow` overwrote the still-unsaved fields with the older server snapshot.
